# 模型分级策略 — model_strategy.md

> Agent 每次生成内容前，根据任务类型选择对应模型。
> 不同任务对"说人话"的要求不同，用错模型是最大的暴露风险。

---

## 核心原则

```
任务越轻量 → 用越小的模型（更口语化，更便宜，更快）
任务越重要 → 用越好的模型（逻辑更强，转化率更高）
永远不要用同一个模型跑所有任务
```

---

## 模型分级表

| 级别 | 任务类型 | 推荐模型 | 备用模型 | 原因 |
|------|----------|----------|----------|------|
| L1 | 养号期浏览/点赞/关注（无文字生成） | 无需模型 | — | 纯操作，不调用 LLM |
| L2 | Tier 1 评论（1-2句，纯互动） | gemini-flash / gpt-4o-mini | deepseek-v3 | 越随意越好，小模型反而更自然 |
| L3 | Tier 2 评论（有建议，需逻辑） | gpt-4o / claude-sonnet | gemini-pro | 需要给出有说服力的建议 |
| L4 | 热点内容快速生成 | gemini-flash / gpt-4o-mini | deepseek-v3 | 速度优先，抢时间窗口 |
| L5 | Tier 3 私信（首次，建立信任） | claude-sonnet / gpt-4o | gemini-pro | 需要情感理解和自然表达 |
| L6 | 对话跟进（Stage 1-3，推进转化） | claude-sonnet | gpt-4o | 需要记忆上下文和策略推进 |
| L7 | 周报生成 / A/B 结果分析 | claude-sonnet / gpt-4o | — | 分析任务，质量优先 |

---

## 模型配置（在 config.md 中设置）

```yaml
models:
  l2_comment: "gemini-flash"        # Tier 1 评论
  l3_comment: "gpt-4o"             # Tier 2 评论
  l4_trending: "gemini-flash"       # 热点快速响应
  l5_dm_first: "claude-sonnet"      # 首次私信
  l6_dm_followup: "claude-sonnet"   # 对话跟进
  l7_analysis: "claude-sonnet"      # 分析周报

  # 备用模型（主力被限流/封号时自动切换）
  fallback:
    tier1: "deepseek-v3"
    tier2: "deepseek-v3"
    tier3: "gpt-4o"
```

## 防封号策略

**为什么 Gemini 会被封：**
- 大量调用 + 内容含"评论""引流""私信"等词 → 触发内容政策
- 单一 API key 调用量过大 → 触发速率限制

**应对方案：**

```yaml
api_rotation:
  # 多个 API key 轮换，避免单 key 超限
  gemini_keys:
    - "key_1"
    - "key_2"
    - "key_3"
  
  # 每个 key 每天最大调用次数（保守值）
  daily_limit_per_key:
    gemini_flash: 500
    gpt4o_mini: 1000
    claude_sonnet: 300

  # 调用间隔（秒），避免触发速率限制
  call_interval_sec: [3, 8]
```

**Prompt 写法规范（避免触发模型内容政策）：**
- ❌ 不在 prompt 里出现"引流""截流""推广""affiliate"等词
- ✅ 用"分享使用经验""回应用户问题""提供产品建议"替代
- ❌ 不描述"批量操作""自动发评论"
- ✅ 描述为"为这位用户的问题写一条真实的回复"

## 每日 Token 消耗估算

> 基于每天 28 次 HEARTBEAT 触发，每次操作 10 条内容

| 任务 | 每条 token | 每天条数 | 每天消耗 |
|------|-----------|----------|----------|
| Tier 1 评论（L2） | ~200 | 140 | ~28,000 |
| Tier 2 评论（L3） | ~500 | 84 | ~42,000 |
| Tier 3 私信（L5） | ~800 | 56 | ~44,800 |
| 对话跟进（L6） | ~1,500 | 20 | ~30,000 |
| 热点响应（L4） | ~300 | 30 | ~9,000 |
| 周报分析（L7） | ~3,000 | 1/7天 | ~430/天 |

## 模型切换规则

Agent 遇到以下情况自动切换到备用模型，并记录到 error_log.md：

| 触发条件 | 处理方式 |
|----------|----------|
| API 返回 429（限流） | 切换到当前模型的下一个 key，等待 60 秒 |
| 所有 key 都 429 | 切换到备用模型 |
| API 返回 5xx | 等待 30 秒后重试，重试 3 次后切换备用 |
| 内容被模型拒绝生成 | 改写 prompt（去掉敏感词），用备用模型重试 |
| 备用模型也失败 | 跳过本条操作，记录日志，继续下一条 |
