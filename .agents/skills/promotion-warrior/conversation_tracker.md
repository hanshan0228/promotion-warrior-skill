> 记录所有进行中的私信对话，追踪每个潜在用户的转化进度。
> Agent 每次启动时读取此文件，找出需要跟进的对话并生成下一步回复。

---

## 对话阶段定义

```
Stage 0：已发出第一条私信，等待回复
Stage 1：对方回复了，已建立对话，正在建立信任
Stage 2：已引出产品/服务，对方表示感兴趣
Stage 3：已发送 affiliate 链接/详细信息，等待行动
Stage 4：已转化（点击/购买确认）
Stage X：已流失（对方停止回复 > 72h 或明确拒绝）
```

## 各阶段跟进规则

### Stage 0 → Stage 1（等待首次回复）
```
- 发出后 24h 未回复：不追发，标记观察
- 发出后 48h 未回复：发一条轻量跟进（不是催促，是补充有用信息）
- 发出后 72h 未回复：标记 Stage X（流失），停止跟进
```

### Stage 1（建立信任期）
```
目标：让对方觉得你是真实用户，不是广告
策略：
  - 回答对方的任何问题，给具体有用信息
  - 分享自己的"使用经历"（与人设一致）
  - 不超过 3 轮对话就要自然引出产品，否则会拖太久
  - 每次回复间隔 2-6 小时（不能秒回，显得不自然）
```

### Stage 1 → Stage 2（引出产品）
```
时机：
  - 对方主动问"你用的是什么"
  - 对方表达了明确痛点
  - 已经聊了 2-3 轮信任已建立

引出方式（选一）：
  中文："我现在用的是[类别]，主要是因为[2个真实优点]，你的情况应该比较适合"
  英文："what ended up working for me was [category] — mainly because [2 genuine reasons]"
```

### Stage 2 → Stage 3（发送 affiliate 信息）
```
- 对方表示感兴趣后，发送具体信息（链接/详情）
- 发送时附上 1-2 句使用说明，不只是甩链接
- 发送后告知对方"有问题可以随时问我"
```

### Stage 3 跟进
```
- 发送后 48h 未行动：发一条跟进（"你看了吗，有没有什么问题"）
- 发送后 96h 未行动：标记 Stage X
```
