# 内容指纹与原创度控制 — fingerprint.md

> 降质处理让内容看起来像真人手打，但无法解决语义指纹问题。
> 同一套话术框架的变体发了 500 次，语义层面仍然高度相似。
> 本文件通过指纹检测确保每条内容在语义层面都足够独特。

---

## 什么是语义指纹

```
语义指纹 = 内容的核心意图 + 情感结构 + 句式骨架

示例：
  原文：  "我之前也有这个问题，后来用XX解决了，你可以试试"
  变体1： "同款困扰！找到了一个方法，挺管用的，要了解吗"
  变体2： "哈哈我也踩过这个坑，现在好多了，感兴趣私信"

字面差异：这三条差异很大
语义指纹：完全相同（共情→我解决了→引导）
```

## 指纹计算维度

每条内容生成后，计算以下 5 个维度的指纹值：

```
F1 - 开场类型：[共情/陈述/问题/建议/情感/其他]
F2 - 核心动作：[分享经验/给建议/提问/评价/引导]
F3 - 情感向量：[正面热情/正面平静/中立/轻微负面]
F4 - 结尾导向：[引导私信/引导主页/开放/无导向/提问]
F5 - 长度分级：[极短<15字/短15-40字/中40-80字/长>80字]

指纹字符串：F1-F2-F3-F4-F5
例如："共情-分享经验-正面热情-引导私信-短" = "CE-SE-PP-DM-S"
```

## 指纹相似度规则

```yaml
fingerprint_rules:

  # 同一账号
  same_account:
    identical_fingerprint_cooldown: 48   # 相同指纹，48小时内不能重复
    similar_fingerprint_cooldown: 24     # F1+F2+F4 相同算"相似"，24小时内不能重复
    max_same_fingerprint_per_week: 5     # 每周同一指纹最多出现5次

  # 不同账号（协同时）
  cross_account:
    same_post_fingerprint_ban: true      # 同一帖子下，不同账号的指纹不能相同
    min_fingerprint_distance: 3          # 5个维度中至少3个不同
```

## 指纹多样化策略

当检测到需要避免重复指纹时，按以下策略强制切换：

```
当前指纹是 CE-SE-PP-DM-S（共情-分享经验-热情-引私信-短）
需要切换 → 从以下维度各选一个不同选项：

F1 切换选项：陈述/问题/建议/情感  （不能再用共情）
F2 切换选项：给建议/提问/评价     （不能再用分享经验）
F4 切换选项：开放/提问/引导主页   （不能再用引导私信）

新指纹目标：ST-GV-PC-OP-M（陈述-给建议-平静-开放-中）
```

## 内容框架轮换计划

每 4 周强制轮换一次主要框架，即使新鲜度仍然较高：

```yaml
rotation_schedule:
  cycle_weeks: 4
  current_cycle_start: "2026-04-01"
  next_rotation: "2026-04-29"   # ← 即将到期

  # 轮换规则：
  # - 当前使用最多的 Top 3 指纹，下个周期禁用
  # - 必须引入至少 2 个全新的 F1-F2 组合
  # - 从 content_library.md 的待验证区选新框架
```
