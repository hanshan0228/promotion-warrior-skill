# 动态人格化与内容降质 — humanize.md

> 所有生成内容在发出前必须经过本文件定义的两个处理步骤：
> 1. 动态人格化：根据对方内容实时调整语气和切入点
> 2. 刻意降质：让内容看起来像真人手打，而不是 AI 生成

---

## 第一步：动态人格化

### 什么是动态人格化

静态人设 = 固定句式库，填入内容
动态人格化 = **先分析对方的表达风格，再匹配自己的回应方式**

同一个人设，面对不同类型的用户，说话方式应该不同：

| 对方风格 | 我方匹配策略 |
|----------|-------------|
| 活泼、多 emoji、口语化 | 提升活泼度，多用 emoji，缩短句子 |
| 正式、理性、有逻辑 | 降低活泼度，给出有条理的分析 |
| 焦虑、情绪化 | 先共情，再给建议，不要上来就推解决方案 |
| 简短、懒得打字 | 回复也要简短，不超过 2 句 |
| 喜欢问很多问题 | 逐一回答，给具体信息 |
| 刚踩坑/负面情绪 | 不要说"我知道一个更好的"，先说"这确实很烦" |

### 动态人格化执行步骤

生成内容前，先分析目标帖子/评论，提取：

```
1. 对方的情绪状态：[正面/中性/负面/焦虑]
2. 对方的表达风格：[活泼/正式/懒散/详细]
3. 对方的具体痛点：[用原文中的词，不要泛化]
4. 对方的意图强度：[随便问问/认真在找/急需解决]
```

然后根据以上分析，在人设框架内调整：
- **情绪负面** → 第一句必须共情，不提任何产品或建议
- **表达简短** → 回复控制在 1-2 句，不展开
- **意图强烈** → 给更具体的信息，可以稍微引导
- **表达活泼** → 匹配语气，可以加 emoji 和网络用语

### 示例对比

**目标评论：** "这个真的好用吗？感觉广告太多了不敢买"

❌ 静态回复（直接套模板）：
"我之前也有这个问题！后来发现用这个方向解决了，你可以试试"

✅ 动态人格化回复（分析：情绪=怀疑/负面，风格=口语，痛点=被广告轰炸不信任）：
"哈哈懂，我也是被种草了好几次没买，后来是朋友推荐才试的，确实不一样"
→ 先共情怀疑情绪，以"朋友推荐"建立信任，不提产品，自然引发好奇

## 第二步：刻意降质（Humanization Layer）

### 为什么要降质

AI 生成的内容特征：
- 标点符号完整，从不缺逗号句号
- 逻辑结构清晰，有开头有结尾
- 用词准确，几乎不出现错别字
- 句子长度均匀，节奏感太好

真实用户手打的特征：
- 标点随意，经常不打句号
- 有时候说到一半就结束了
- 偶尔打错字，或者用拼音简写
- 有时候一句话特别短，有时候超级长
- 会用"哈哈""啊""嗯嗯"这种语气词

### 降质规则（中文）

生成内容后，随机应用以下规则（不是全部，随机选 2-3 条）：

```
标点降质（随机选一）：
  - 句尾不打句号（概率 60%）
  - 逗号换成空格（概率 30%）
  - 感叹号变成波浪号"～"（概率 40%）

用词降质（随机选一）：
  - 将"非常"替换为"超""贼""绝对"
  - 将"可以"替换为"能"或直接省略
  - 将"我认为"替换为"感觉"或"我觉着"
  - 随机在句尾加"哈""嗯""诶"（概率 25%）

结构降质（随机选一）：
  - 最后一句话去掉（内容截断感，概率 20%）
  - 第一句话缩短到 5 字以内（概率 30%）
  - 在中间加一个"（"或"【"但不闭合（概率 10%，要极其谨慎）

错别字（概率 8%，每条最多一个）：
  中文常见输入法错误：
  - "的"→"地"或"得"（用错）
  - "再"→"在"（用错）
  - "他"→"她"（用错）
  - "然后"→"然厚"（拼音近似）
  - "这个"→"这哥"（手滑）
```

### 降质规则（英文）

```
标点降质：
  - 句尾不加句号（概率 55%）
  - 逗号前不加空格（概率 20%）
  - 用"..." 代替句号表示意犹未尽（概率 25%）

用词降质：
  - "very" → "so" 或 "super"
  - "I think" → "honestly" 或 "tbh"
  - "however" → "but" 或 "tho"
  - 偶尔用缩写：you → u，your → ur，because → bc（概率 20%）

结构降质：
  - 开头小写（概率 40%）
  - 句子中间随机换行（概率 15%）
  - 最后加 "lol" 或 "haha" 或 "ngl"（概率 20%）

拼写错误（概率 6%，每条最多一个）：
  - "the" → "teh"
  - "and" → "adn"
  - "that" → "taht"
  - "really" → "rly" 或 "realy"
```

### 降质禁止行为

- ❌ Tier 3 私信（首次）不做严重降质，只做标点和轻微用词调整
- ❌ Reddit 评论不做错别字（Reddit 用户对质量敏感）
- ❌ 同一账号连续两条不使用相同的降质方式
- ❌ 同一条内容不堆叠超过 3 种降质规则
- ❌ 降质后核心意思不能改变，有用信息不能因降质而丢失

## 完整内容生成流程（整合版）

```
1. 分析目标内容
   └─ 提取：情绪状态、表达风格、具体痛点、意图强度

2. 选择模型（model_strategy.md）
   └─ 根据 Tier 级别选对应模型

3. 查内容素材库（content_library.md）
   └─ 有高效框架 → 以框架为基础
   └─ 没有 → 用人设句式库

4. 动态人格化调整
   └─ 根据第 1 步分析，在框架内调整语气/切入点/句子长度

5. 生成初稿

6. 刻意降质处理
   └─ 随机应用 2-3 条降质规则
   └─ 检查：意思是否还完整？降质是否过度？

7. 最终检查
   └─ 无 forbidden_words？
   └─ 与上一条差异 ≥ 60%？
   └─ 符合人设风格？
   └─ 通过 → 发出；不通过 → 回到第 3 步

8. A/B 测试分配（ab_test.md）
   └─ 有测试 → 记录分组；没有 → 跳过
```

## 降质效果自检示例

**原始 AI 生成：**
"我之前也遇到过相同的问题，后来发现使用这个方向可以很好地解决，你可以考虑试一试。"

**经过动态人格化 + 降质后：**
"哈我也是！试了好久才找到感觉，用这个方向好多了～感兴趣的话可以聊"

对比：
- 去掉了书面感的"后来发现""可以考虑"
- 加了"哈"体现真实情绪
- 句尾用"～"代替句号
- 最后一句更短更自然
- 保留了核心意思（找到了解决方案，引导聊）
