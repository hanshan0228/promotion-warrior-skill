# 账号间知识共享 — knowledge_sharing.md

> 账号 A 发现了一条高效评论，账号 B 不应该重新发明轮子。
> 本文件让各账号的成功经验在系统内流通，但必须经过改写才能跨账号使用。

---

## 共享的三类知识

```
类型 1：高效内容模式（回复率 > 15% 的内容特征）
类型 2：高意向用户特征（哪类帖子/评论下的用户更容易转化）
类型 3：平台行为规律（哪个时段、哪类内容、哪个话题互动最好）
```

---

## 知识提炼规则

每周日，从各账号的 performance 数据中提炼知识：

```
触发条件：单条内容回复率 > 15%，或某类帖子转化率 > 30%

提炼内容（不记录原文，只记录模式）：
  - 什么切入角度
  - 什么情感基调
  - 什么内容结构
  - 目标用户的什么特征触发了高回复

禁止记录：
  - 原始评论文字（防止其他账号直接复用被识别）
  - 用户的个人信息
```

## 知识应用规则

账号在生成内容时，读取此文件中与当前平台和 Tier 匹配的知识：

```
1. 查找 platform + tier 匹配的高效模式（知识库 → 高效内容模式）
2. 找到匹配项 → 将模式特征作为生成约束，而不是直接复制
3. 生成后必须通过指纹检查（fingerprint.md），确保与原始内容有足够差异
4. 查找目标用户是否符合高意向特征（知识库 → 高意向用户特征）
5. 符合 → 优先进入 Tier 2，提高用户评分（user_scoring.md）
```
