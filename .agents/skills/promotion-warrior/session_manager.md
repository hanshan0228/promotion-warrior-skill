## Session 验证流程

**每次启动时（自动执行）：**

```
1. 读取所有账号的 expires_at
2. 计算距今天数
3. < 预警天数 → 在启动报告中标红提示用户
4. 已过期 → 将账号标记为 needs_login，跳过本次操作
5. 对本次要使用的账号执行轻量验证：
   访问平台首页，检查是否仍处于登录状态
   → 仍登录：更新 last_check 时间
   → 已退出：标记 needs_login，通知用户
```

**Session 续期方法：**

```
自动续期（部分平台支持）：
  - 每次操作成功 = Cookie 自动续期
  - 只要账号保持活跃操作，Cookie 通常不会主动失效

手动续期（用户操作）：
  1. 用 openclaw browser 打开对应平台
  2. 手动登录账号
  3. 登录成功后 openclaw browser 自动保存新 Cookie
  4. 在此文件更新 login_time 和 expires_at
  5. 将账号状态改回 active
```

## 批量过期风险预防

**问题：** 如果所有账号同一天登录，就会同一天过期，导致系统停摆。

**解决方案：** 错峰登录策略

```
账号登录时间间隔建议：
  同一平台的账号：间隔 ≥ 7 天登录
  不同平台的账号：间隔 ≥ 2 天

实现方式：
  新增账号时，查看此文件中同平台的 login_time
  选择距离最远的时间点登录新账号
```
