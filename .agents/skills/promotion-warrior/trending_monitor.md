# 热点感知 — trending_monitor.md

> Agent 每次启动时扫描热点，发现与 niche 相关的热点立即插队执行，不等下次 HEARTBEAT。
> 热点窗口通常只有 2-4 小时，必须快速响应。

---

## 热点扫描来源

每次启动时扫描以下来源，寻找与 `config.md → niche` 相关的热点：

### 中文平台热点源
```yaml
sources:
  weibo_hot: "https://s.weibo.com/top/summary"          # 微博热搜榜
  xiaohongshu_hot: "小红书搜索框热搜词"                   # 搜索时下拉热词
  douyin_hot: "抖音热点榜"                               # 抖音发现页热榜
  baidu_hot: "https://top.baidu.com/board?tab=realtime" # 百度实时热点
```

### 英文平台热点源
```yaml
sources:
  twitter_trending: "X 趋势话题（地区设为目标市场）"
  reddit_rising: "目标 subreddit 的 rising 页面"
  google_trends: "https://trends.google.com/trending"
  tiktok_discover: "TikTok 发现页热门话题"
```

## 热点相关性判断规则

扫描到热点后，判断是否值得介入：

```
相关性评分标准（满足越多分越高）：
  +3 分：热点话题与 niche 直接相关（同品类/同场景）
  +2 分：热点中有用户表达购买需求/求推荐
  +2 分：热点是负面评价竞品（截流好时机）
  +1 分：热点是 niche 领域大 V 发帖引发讨论
  +1 分：热点带有明确的消费场景（节日/季节/事件）
  -2 分：热点话题敏感或有争议风险
  -3 分：热点与 niche 牵强相关，强行蹭显假

评分 ≥ 4 分 → 立即插队执行热点任务
评分 2-3 分 → 加入本次普通任务队列
评分 < 2 分 → 忽略
```

---

## 热点介入策略

**评分 ≥ 4 分时的执行流程：**

1. 暂停当前计划中的普通任务
2. 在热点话题/帖子下执行 Tier 1（快速占位曝光）
3. 找出评论区含 intent_signals 的用户执行 Tier 2
4. 热点任务完成后恢复普通任务队列
5. 整个热点窗口操作时间控制在 2 小时内

**热点评论内容要求：**
- 必须与热点话题直接相关，不能硬蹭
- Tier 1 评论要回应热点本身，不提产品
- Tier 2 才可以自然引出 niche 相关信息
